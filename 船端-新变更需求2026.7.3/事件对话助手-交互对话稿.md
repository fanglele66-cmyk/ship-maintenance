## 第四阶段：逐项排查循环（核心交互区）

> **结构说明（对照操作手册）：**
> - 排查项（T1-T4）→ 步骤（steps）→ 关键点（items）
> - **反馈粒度：关键点级**，每个关键点独立标记 ✓正常 / ⚠异常（+描述）
> - 步骤结果：由该步下所有关键点汇总（有异常→⚠，全部正常→✅）
> - 排查项结果：由所有步骤汇总

## 附：排查反馈结构对照表（操作手册对齐说明）

> 本节说明对话稿中排查反馈的层级结构如何对齐操作手册的数据模型。

### 操作手册数据结构

```
todos.inspections[]              ← 排查项（T1-T4）
  ├─ id, title, defaultFold
  ├─ result: null | "normal" | "abnormal"    ← 排查项级结果（由步骤汇总）
  ├─ detail, operator, time
  └─ steps[]                                    ← 步骤
       ├─ title
       ├─ items[]                                ← 关键点（检查清单项）
       │    └─ "目视检查油箱外壁焊缝处有无渗漏油渍"
       ├─ time
       └─ hasImages: boolean
```

### 对话稿反馈层级映射

```
排查项 T1 · 液压系统故障排查
  │
  ├─ 步骤 1/6 · 检查液压油箱外观及液位计
  │    ├─ 关键点：目视检查油箱外壁焊缝处有无渗漏油渍  → [✓正常]
  │    ├─ 关键点：检查油箱底部放油堵有无湿润痕迹      → [⚠异常] + 描述
  │    ├─ 关键点：对比液位计读数与远程监控值是否一致    → [✓正常]
  │    └─ 关键点：检查液位计上下连通阀是否处于全开位置  → [✓正常]
  │    └─ 步骤结果：⚠️ 有异常（1/4 关键点异常）
  │
  ├─ 步骤 2/6 ~ 6/6 ...
  │
  └─ 排查项结果：⚠️ 异常（3/6 步骤异常）
     → 调用 updateInspection(eventId, "T1", "abnormal", detail, operator)
     → 自动添加日志：{stage:"排查", title:"排查项反馈：液压系统故障排查", ...}
```

### 反馈粒度对比

| 维度 | v3（旧版） | v4（本版，对齐操作手册） |
|------|-----------|----------------------|
| 反馈层级 | 排查项级（平铺下拉） | **关键点级**（步骤下逐项标记） |
| 反馈结构 | 滤器压差 [正常/偏高/偏高很多] | 步骤1→关键点1 [✓正常/⚠异常] + 描述 |
| 步骤对应 | 步骤与反馈项不对应 | 每个步骤下的关键点即为反馈项 |
| 结果汇总 | 无汇总，手动选整体结果 | 步骤结果由关键点自动汇总，排查项结果由步骤自动汇总 |
| 数据来源 | 虚构的冷却水场景 | 操作手册 EVT-2026-0042 真实 Mock 数据 |
| 步骤数量 | 每项 4 步（简单） | T1: 6步/21关键点, T2: 4步/15关键点, T3: 4步/15关键点, T4: 5步/19关键点 |
| 拍照标记 | 无 | 步骤4(T1)、步骤5(T4)标记 📷 hasImages |
