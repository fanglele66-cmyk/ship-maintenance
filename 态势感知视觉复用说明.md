# 态势感知视觉复用V1.0说明

> **日期**：2026-07-03  
> **核心原则**：态势感知 = V1.0 首页 + V1.0 监控中心的拼接，加上右侧常驻助手

---

## 一、复用策略

**不重新设计任何视觉元素**，直接复用 V1.0 的：

1. **首页（HomeView.vue）的视觉元素**：
   - 船体线框图 + 扫描动效
   - 悬浮设备卡片（毛玻璃 + 青色边框）
   - 船舶信息卡（航行状态/健康分/下次维保）

2. **监控中心（MonitorView.vue）的视觉元素**：
   - 系统筛选标签栏
   - 传感器卡片网格（左侧彩条 + 大数值 + 状态徽章）
   - 趋势图（ECharts 折线图 + 多线对比 + 时间范围切换）

## 二、三层递进展示

### 第一层：船体线框图 + 悬浮设备卡片

**来源**：V1.0 HomeView.vue

**复用内容**：
- `shipBg.js` 导出的 base64 透底 PNG（960×640）
- 扫描动效（scan-line + scan-glow + scan-grid-overlay）
- 4 个设备浮动卡片（主机/锅炉/舵机/空压机）
- 卡片定位坐标（主机63%,60% / 锅炉70%,36% / 舵机91%,62% / 空压机35%,54%）
- 卡片视觉规格（毛玻璃 + 青色边框 + 状态圆点）

**交互变化**：
- V1.0：点击设备卡片 → 多选过滤
- V2.0：点击设备卡片 → 进入第二层（传感器网格）

**右侧助手**：
- 显示船舶概览（健康分 + 设备异常摘要 + 重点关注建议）
- 跟随用户点击设备 → 切换上下文

---

### 第二层：系统筛选标签 + 传感器卡片网格

**来源**：V1.0 MonitorView.vue

**复用内容**：
- 系统筛选标签栏（全部/燃油/排气/启动阀组...）
- 传感器卡片网格（3列）
- 每张卡片：
  - 左侧 3px 彩条（超限红/异常橙/正常绿）
  - 传感器英文名（粗体）
  - 中文名（小字）
  - 所属系统·设备名（小字）
  - 当前值（大号 Consolas，颜色随状态）
  - 单位（小字）
  - 状态徽章（右下角）
- 超限/异常传感器自动排在最前面

**交互变化**：
- V1.0：点击传感器卡片 → 多选，趋势图显示多条曲线
- V2.0：点击传感器卡片 → 进入第三层（趋势图详情）

**右侧助手**：
- 显示该系统传感器概况（几个超限/预警，优先关注哪个）
- 跟随用户点击传感器 → 切换上下文

### 第三层：左侧传感器列表 + 右侧趋势图

**来源**：V1.0 MonitorView.vue（点击传感器后的详情视图）

**复用内容**：
- 左侧传感器列表（宽度 200px）
- 每个列表项：
  - 左侧 3px 彩色竖条
  - 传感器英文名（粗体）
  - 中文名（小字）
  - 当前值（大号 Consolas）
  - 状态徽章
- 右侧大趋势图（ECharts）：
  - 蓝色曲线 + 深色背景
  - 网格线 + Y轴刻度 + X轴时间
  - 多线对比开关
  - 时间范围按钮（1H/6H/24H/7D/自定义）
  - AI 分析摘要（右下角，可收起）

**交互变化**：
- V1.0：点击左侧列表项 → 趋势图切换到该传感器
- V2.0：同 V1.0（无变化）

**右侧助手**：
- 显示趋势解读（最近变化 + 可能原因 + 是否建议生成事件快照）
- 跟随用户切换传感器 → 自动更新解读

## 三、层级切换动画

| 切换 | 动画 | 时长 | 缓动函数 |
|------|------|------|---------|
| 第一层→第二层 | 船体图向左滑出 + 传感器网格从右滑入 | 0.35s | `cubic-bezier(0.4,0,0.2,1)` |
| 第二层→第三层 | 传感器网格向左滑出 + 趋势图左右分栏从右滑入 | 0.35s | `cubic-bezier(0.4,0,0.2,1)` |
| 返回上一层 | 当前层向右滑出 + 旧层从左滑入 | 0.35s | `cubic-bezier(0.4,0,0.2,1)` |

**触发方式**：
- 点击卡片（进入下一层）
- 点击 ← 返回按钮
- 左边缘右滑（手势）
- 双指捏合（快速返回上一层）

**层级指示器**：
- 屏幕顶部居中
- 3 个圆点/短横线
- 当前层高亮：`#1890FF`，宽度 18px（药丸形）
- 其他层灰色：`#1E3A5F`，宽度 6px（圆点）

## 四、实现方式

### 方案 A：组件拆分（推荐）

```
SituationView.vue（容器）
├── ShipOverview.vue（第一层：船体图 + 设备卡片）
├── SensorGrid.vue（第二层：系统标签 + 传感器网格）
├── SensorTrend.vue（第三层：传感器列表 + 趋势图）
└── AssistantPanel.vue（右侧常驻助手）
```

**优点**：
- 每个层级独立组件，便于维护
- 可以直接复制 V1.0 的 HomeView.vue 和 MonitorView.vue 中的代码
- 动画过渡清晰

### 方案 B：单文件（不推荐）

```
SituationView.vue
├── <div class="layer-1" v-show="currentLayer === 1">...</div>
├── <div class="layer-2" v-show="currentLayer === 2">...</div>
└── <div class="layer-3" v-show="currentLayer === 3">...</div>
```

**缺点**：
- 文件过大（超过 1000 行）
- 难以维护

## 六、助手跟随逻辑

```javascript
// 态势感知助手上下文管理
const situationContext = ref({
  layer: 1,           // 当前层级
  deviceId: null,     // 当前选中的设备ID
  sensorId: null,     // 当前选中的传感器ID
  system: null        // 当前系统名称
})

// 监听层级变化
watch(() => situationContext.value.layer, (newLayer) => {
  if (newLayer === 1) {
    // 第一层：显示船舶概览
    pushShipOverview()
  } else if (newLayer === 2) {
    // 第二层：显示该系统传感器概况
    pushSystemSensorSummary(situationContext.value.deviceId)
  } else if (newLayer === 3) {
    // 第三层：显示趋势解读
    pushSensorTrendAnalysis(situationContext.value.sensorId)
  }
})

// 监听设备切换
watch(() => situationContext.value.deviceId, (deviceId) => {
  if (deviceId) {
    pushSystemSensorSummary(deviceId)
  }
})

// 监听传感器切换
watch(() => situationContext.value.sensorId, (sensorId) => {
  if (sensorId) {
    pushSensorTrendAnalysis(sensorId)
  }
})
```

## 七、总结

**态势感知 = V1.0 首页 + V1.0 监控中心 + 右侧常驻助手**

**不重新设计任何视觉元素**，直接复用 V1.0 的代码和样式。

**唯一的变化**：
- 布局从三栏（左监控+中船舶+右检修）改为两栏（左侧信息+右侧助手）
- 交互从平铺改为三层递进（船体→传感器网格→趋势图）
- 助手从独立面板改为 Tab 内常驻
- 层级切换用滑动动画
