核心：快照 vs 实时
这是 V2.0 最关键的数据区分，两个 Tab 的数据性质完全不同。
📸 事件中心 = 快照
定义：传感器检测到异常那一刻，系统把当时的读数"拍一张照片"存下来。

特点：
  • • 数据不会变——事件发生时是 92.4°C，永远就是 92.4°C
  • • 用户看事件详情时，看到的是"事发当时的数据"
  • • 即使现在水温已经降下来了，事件记录里还是当时的快照

    • 就像监控摄像头抓拍的一张照片。
    📡 态势感知 = 实时
    定义：传感器当前的实时读数，持续更新。

    特点：
  • • 数据一直在变——和船上仪表盘看到的一样
  • • 用户看到的是"现在设备在干什么"
  • • 如果刚才的事件已经处理完了，实时数据可能已经恢复正常

    • 就像监控摄像头的实时画面。
    布局模式汇总
    每个 Tab 的布局方式和交互规则。
    📌 事件中心
    默认：双栏（事件列表 + 通用助手）
    选中事件：左栏收起 → 产物抽屉滑入（查看器 + 事件助手）
    数据：📸 快照（事件发生时冻结）
    助手角色：主工作区（所有操作在此完成）
    📡 态势感知
    布局：双栏（设备树 + 实时看板）
    交互：查看全局设备状态、实时数据
    数据：📡 实时（每秒刷新）
    助手：支持问答（"这台设备现在怎么样？"）
    📖 知识库
    布局：双栏（分类目录 + 内容列表）
    交互：浏览、搜索、查看文档
    数据：静态文档
    助手：事件模式下自动检索引用
    🚫 已移除：个人中心
    理由：一船只有一个 pad，谁都用这个 pad，不需要个人账号。
