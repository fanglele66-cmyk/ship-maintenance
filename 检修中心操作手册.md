## 一、系统概述

「船舶设备检修助手」是一个面向船舶轮机管理的智能化检修平台，核心功能模块包括：
- **检修中心（#/repair）** — 事件管理、排查流程、维修作业、AI分析
- **设备监控** — 实时传感器数据、设备状态、趋势图
- **知识库** — 79份技术文档（设备手册/操作规程/故障案例/技术通告）
- **日检报告** — 每日巡检结果与AI健康评估

---

## 二、检修中心 — 整体结构

检修中心是系统的核心工作模块，采用**事件驱动**的工作流：

```
事件触发 → 确认阶段 → 排查阶段 → 维修阶段 → 归档
```

1. 立即补充液压油至正常油位（≥60%），优先选用 ISO VG 46 抗磨液压油
2. 对液压管路进行全面目视检查，重点检查各接头、法兰、软管段的渗漏痕迹

**工程背景知识**：

> - 油量减少 → 散热能力下降 → 油温升高 → 内泄漏增加
>

#### 3.1.2 排查任务清单（4项）

**T1 - 液压系统故障排查**（默认展开）
| 步骤 | 检查项 |
|------|--------|
| ① 检查液压油箱外观及液位计 | 目视检查油箱外壁焊缝处有无渗漏油渍；检查油箱底部放油堵有无湿润痕迹；对比液位计读数与远程监控值是否一致；检查液位计上下连通阀是否处于全开位置 |
| ② 记录当前油位和油温初始数据 | 拍照记录液位计当前读数；记录液压油温度（正常范围40~70℃）；读取系统压力表稳态值（正常16~21MPa）；记录电机运行电流 |
| ③ 目视检查管路各接头及软管段 | 重点检查高压管路法兰接头有无油渍；检查软管段有无鼓包/老化/龟裂/渗油；检查管路支架/卡箍处有无摩擦磨损痕迹；检查弯头、三通等应力集中部位有无湿润 |
| ④ 使用荧光检漏剂定位微漏点 📷 | 在油液中按规定比例加入紫外荧光示踪剂；系统循环运行15min使示踪剂充分扩散；在疑似渗漏区域用UV灯照射观察荧光痕迹；标记所有发现的微漏点位置并拍照记录；评估每个泄漏点的严重程度 |
| ⑤ 检查液压泵吸入口状态 | 听诊泵吸入口有无气蚀噪声（嘶嘶声）；检查吸入滤器压差是否异常增大；观察泵体有无异常温升 |
| ⑥ 取样化验液压油品质 | 在运行状态下从取样阀放油冲洗3倍管容积；采集500ml油样装入洁净取样瓶；送实验室检测含水量/颗粒度/粘度指标；对比上次化验结果评估油品劣化趋势 |

**T2 - 系统或油柜吸口堵塞排查**（默认折叠）
| 步骤 | 检查项 |
|------|--------|
| ① 检查吸油口滤网状态 | 拆检吸口滤器有无堵塞/异物附着；记录滤网表面杂质类型和沉积量；评估滤网通流面积减少百分比；检查滤网骨架有无变形/破损 |
| ② 测量吸口负压值 | 在额定流量下读取吸入侧真空表数值；对比历史正常负压基准判断有无异常阻力；在不同转速工况下记录负压变化趋势；若负压超过-0.08MPa需立即清洗滤器 |
| ③ 检查油箱底部沉积物 | 打开放油旋塞取样底部油液；观察有无金属屑/水分/絮状物；用磁铁检查铁磁性颗粒含量；评估是否需要彻底清洗油箱内部 |
| ④ 检查回油过滤器状态 | 读取回油滤器压差指示器状态（绿/黄/红）；检查回油管路背压是否偏高；判断是否需更换回油滤芯 |

**T3 - 压力传感器故障排查**（默认折叠）
| 步骤 | 检查项 |
|------|--------|
| ① 校验压力传感器输出信号 | 在管路同一测点安装标准压力表；交叉比对传感器读数与标准表偏差；在全量程范围内多点校验（25%/50%/75%/100%）；检查传感器接线端子有无松动/腐蚀 |
| ② 检查传感器供电与信号回路 | 测量供电电压是否在额定范围（24V DC ±5%）；用万用表检测信号输出（4-20mA）线性度；检查回路保险丝/断路器状态；检查信号隔离器/安全栅是否正常 |
| ③ 排查电磁干扰源 | 检查传感器线缆屏蔽层接地是否可靠；评估附近大功率设备启停是否引起读数跳变；检查线缆布线是否与动力电缆保持间距；必要时加装信号滤波器 |
| ④ 检查传感器机械安装 | 确认传感器与测压孔的密封连接无泄漏；检查测压导压管有无堵塞/弯折；确认阻尼孔（脉冲衰减器）无堵塞 |

**T4 - 系统管线泄漏排查**（默认折叠）
| 步骤 | 检查项 |
|------|--------|
| ① 分段检查高压管路 | 沿管路走向逐段目视检查接头、焊缝有无油渍；重点检查弯头、三通等应力集中部位；检查管路固定支架处有无磨损渗漏；检查法兰密封垫片处有无湿润痕迹 |
| ② 检查执行机构密封 | 观察液压缸活塞杆密封圈有无渗油；检查液压马达轴封处有无湿润痕迹；操作液压缸全行程观察有无外泄漏加剧；检查伺服阀/比例阀安装面有无渗油 |
| ③ 检查控制阀组内泄 | 在保压状态下测量阀组回油口泄漏量；判断溢流阀/换向阀有无异常内泄；检查阀组安装螺栓力矩是否达标；检查阀块O型圈通道密封有无失效 |
| ④ 检查油箱及附件密封 | 检查油箱呼吸阀/通气孔是否通畅；检查油箱加油口盖密封圈是否完好；检查液位计法兰接头有无渗漏；检查油箱放油堵磁性感温塞有无松动 |
| ⑤ 记录并拍照 📷 | 对每个可疑泄漏点拍照标记坐标位置；在管路图上标注所有泄漏点位置；按严重程度分级（严重/中等/轻微）；整理泄漏点清单供维修参考 |

#### 3.1.3 维修方案（2项）

**R1 - 补充液压油至正常油位（≥60%）**

#### 9.5.1 单项排查交互流程

## 十、维修步骤模板库

系统内置通用维修步骤模板：

### 10.1 密封件更换
1. 拆卸设备相关部位，取出受损密封件
2. 清洁密封槽，检查密封面有无损伤
3. 选用同规格新密封件，表面涂润滑脂后装入
4. 按规定力矩对角均匀拧紧

### 10.2 滤器清洗
1. 隔离滤器进出口阀门
2. 取出脏堵滤芯，清洗或更换新滤芯
3. 清除滤器壳体内部沉积物
4. 复装后恢复投运，确认压差降至正常值

### 10.3 管路泄漏修复
1. 根据排查标记定位泄漏点
2. 隔离并泄压相关管段
3. 更换密封件或补焊修复
4. 复装后保压30min验证无渗漏

### 10.4 参数调整
1. 备份当前参数
2. 按推荐值范围逐步调整参数
3. 在不同工况下验证调整效果

### 10.5 润滑保养
1. 清除旧润滑脂，清洗轴承内部
2. 填入新润滑脂至轴承腔容积的1/2~2/3
3. 检查轴承游隙和转动灵活性

### 10.6 通用维修
1. 按设备标准作业程序执行维修
2. 维修后清理现场、复装部件
3. 设备维修后空载试运行30min，确认无异响/异振/异温
4. 带载运行验证，各项参数达标
5. 记录最终数据并归档

## 十六、润滑油技术规格（知识库参考文档）

### 第12章 发动机润滑油选用指南

#### 12.2 润滑油分类与牌号

| 类别 | 牌号 | 粘度等级 | 适用机型 |
|------|------|---------|---------|
| 系统油 | SAE 30 | 100 cSt @ 40°C | MAN 6L48/60CR |
| 系统油 | SAE 40 | 140 cSt @ 40°C | Wärtsilä 6L32 |
| 气缸油 | SAE 50 | 210 cSt @ 40°C | MAN 6L48/60CR |
| 气缸油 | BN 70 | — | MAN 5L35MC |
| 增压器油 | SAE 15W-40 | 110 cSt @ 40°C | ABB TPL 系列 |

#### 12.3.1 碱值（BN）选择

| 燃油硫含量 | 推荐碱值 BN |
|-----------|------------|
| ≤ 0.5% | BN 25-30 |
| 0.5% - 1.5% | BN 40-50 |
| 1.5% - 3.5% | BN 70-80 |
| > 3.5% | BN 100 |

> ⚠ 注意：长期使用低硫燃油（≤0.1%）时，若仍使用高碱值气缸油（BN 70+），可能导致缸套表面抛光（Bore Polishing）。建议切换为 BN 25-30 低碱值气缸油。

#### 12.4 更换周期

| 润滑油类型 | 正常更换周期 | 缩短周期条件 |
|-----------|------------|-------------|
| 系统油 | 每 8,000-10,000 运行小时 | 粘度变化>±20%、水分>0.2%、总碱值下降50% |
| 气缸油 | 持续供油（按油耗） | 残碱值<25 mg KOH/g、油耗增加时 |
| 增压器油 | 每 3,000-4,000 运行小时 | 油品劣化、粘度异常 |

#### 12.5 油品检测指标

| 检测项目 | 正常范围 | 警告值 | 停机值 |
|---------|---------|--------|--------|
| 40°C 运动粘度 | 100-140 cSt | <90 或 >150 | <80 或 >160 |
| 水分含量 | <0.1% | 0.1-0.2% | >0.2% |
| 总碱值 TBN | >20 mg KOH/g | 10-20 | <10 |
| 闪点 | >220°C | 190-220°C | <190°C |
| 不溶物 | <0.5% | 0.5-1.0% | >1.0% |

> ⚠ 警告：当水分含量超过0.2%时，润滑油乳化风险急剧增加。应立即停机检查冷却系统是否存在泄漏。

### A.5 事件来源与优先级体系

**事件来源**：

| 来源 | 触发方式 | 说明 |
|------|---------|------|
| ai（AI检测引擎） | 传感器数据异常 → AI自动分析 → 自动创建事件 | 全程自动，从告警到方案生成 |
| inspection（巡检发现） | 人工巡检发现异常 → 手动录入系统 | 巡检人员通过移动端或终端上报 |
| crew（船员上报） | 船员目视发现异常 → 通过对话/表单上报 | 非专业人员的初步报告 |

**优先级定义**：

| 优先级 | 颜色 | 含义 | 响应要求 |
|--------|------|------|---------|
| high（高） | 🔴 红色 | 影响安全或可能导致停机 | 立即响应，30分钟内确认 |
| medium（中） | 🟡 黄色 | 性能下降或潜在风险 | 2小时内确认 |
| low（低） | 🟢 绿色 | 计划性维护或轻微异常 | 当日内确认 |

### A.6 多用户协同

检修中心支持多人同时操作：

| 场景 | 机制 |
|------|------|
| 同一事件多人查看 | 产物区实时同步，所有用户看到相同内容 |
| 排查反馈提交 | 提交后所有用户可见，其他用户的排查项自动更新状态 |
| 维修步骤执行 | 每步记录操作人和时间，可追溯 |
| 事件状态变更 | 所有在线用户同步收到通知 |
| 冲突处理 | 后提交者会看到前者的反馈，系统提示"已有其他人员提交" |
